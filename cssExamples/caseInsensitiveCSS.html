CSS is not case sensitive when it comes to specifying property names. Uncomment the embedded stylesheet rule in the head section to see that even though we used property names like "backGROUND-COLOR" and "COLor", the property names are still processed correctly.

CSS is case-insensitive when it comes to specifying element selectors, but is case sensitive when specifying class and id selectors. For example, the following two element selectors will all work. Notice we are using 'P' and not 'p' - either one will work. 

    P {
        background-color: white;
    }

Notice we're using Div in the style below. We could have used div, DIV, diV, dIV, etc. Anyone of them would have worked.

    Div {
        margin-top: 20px;
    }

However, it is case-sensitive when it comes to class selectors and id selectors. The class names must match, and the id selectors must match as well. Consider the snippet below: 

    mytable {
        border: 1px solid black;
    }
    ...
    <table class="myTable">
This will not work as intended because "mytable" does not match "myTable". The table will not have a solid black border 1px wide.

Consider also the snippet below: 

    #ChemData {
        float: left;
    }
    ...
    <div id="chemData">
This will not work because "ChemData" does not match "chemData". So the div tag with id="chemData" will notk be floated left.
