The Semantic Elements

The semantic elements in HTML5 are the elements that have meaning in and of themselves. For example, the div element is not a semantic element. It is a division tag, so its purpose really is just to display a block of content in a rectangular portion of the screen. The content within the division element could be anything.

However, semantic elements are meant to contain specific types of information. For example, the nav element is meant to contain navigation-related elements - that is, hyperlinks that will help the user navigate the website.

The semantic elements in HTML5 are listed below:

  • article
  • aside
  • details
  • figcaption
  • figure
  • footer
  • header
  • main
  • mark
  • nav
  • section
  • summary
  • time

The header, footer, main, and aside are used similarly. They are used to create the "structure" of a webpage. The header goes at the top, the footer goes at the bottom. In between the header and footer goes the main, and beside the main goes the aside (assuming an aside is used). You can treat these elements like div elements; that is, you can style them like you would a div element.

The nav element is used to contain anchor tags that are used for navigation. Not all hyperlinks are used for navigation purposes. For example, some anchor elements use the mailto: protocol to indicate that the hyperlink is an e-mail address, so that clicking on it will probably bring up the default e-mail application. Some anchor elements use the tel: protocol, indicating that the hyperlink is a phone number. Clicking on this hyperlink will bring up a phone dialer app (in particular on smartphones) to dial the phone number.

The details and summary elements are used together. The details element contains the summary element, and under the summary element appears more text (or images). The summary is used to give the details block a summary, caption, or title. When the summary is clicked, the rest of the details block is alternatively displayed and hidden.

The figure and figcaption elements are typically used together. You create a figure element. Inside the figure element you use an img element. Above or below the img element you use a figcaption element to give the figure a caption.

The mark and time have somewhat similar purposes. Previously, many webpage developers used a span element to do what the mark and time elements now do. The time element is used to contain a time value (for events like a theatre showing time, or any scheduled time like class time, doctor's appointment time, etc.). It is not where the user is asked to enter a time, it is where the webpage developer is telling anyone reading the webpage (anyone including programs that read the webpage) that the contents of the time element contain a time value.

article

Article elements are used to enclose content that stands alone and could be syndicated (i.e., sold as content to other webpages). This could be, for example, a newspaper article, a science article, a math article, a blog article, a forum post, etc.

Examples using the article element

Group Theory for the Common Man

Group theory is a subject area of math which first appeared around the time of the French Revolution. In fact, its birth was given by a young man named Evariste Galois, who took part in the French Revolution. Unfortunately, Evariste died very young as the result of a duel.

Group theory is all about groups and their properties. A group is defined as a set G of elements, and an associative operation *:GxG -> G which you can consider to be "multiplication" or "addition" of the elements. This set G must contain an element which behaves like a zero element. That is, each group G must contain an element g such that for every element g' in G, we have g' + g = g', and g + g' = g'. The operation must be associative means that for all elements g, g', and g'' of G we have (g * g')*g'' = g*(g' * g'')

You are already familiar with some groups. Consider the set Z of integers, with the operation of addition. That is a group. The operation of addition is associative, because for any integers a, b and c we have (a + b) + c = a + (b + c). Also, it contains a zero element, the number 0. This zero element satisfies the property 0 + g = g for all g in Z and g + 0 = g for all g in Z.
