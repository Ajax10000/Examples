Input Element

Many different types of input fields can be created with the input element. Exactly which type of input field is created depends on the type attribute you use. The type attribute can have the following values:
button - to allow the developer to display a button. Often used to call a JavaScript function after the user has filled in the appropriate fields.
checkbox - to allow the user to click on one or more check boxes
color - to allow the user to click on a button that brings up a dialog box that allows the user to select a color. After the user selects a color, the color of the button is the color the user selected.
date - to allow the user to enter a date
datetime-local - to allow the user to enter a date and time, but without specifying a time zone
email - to allow the user to enter an e-mail address
file - to allow the user to upload a file
hidden - to allow the developer to store extra hidden information within a form. Often this hidden information is information the user filled out in a previous form.
image - to allow the user to use an image as a submit button
list - to allow the user to select an value from a dropdown list or enter any value
month - to allow the user to enter a month
number - to allow the user to enter a number
password - like a text field, but the user will typically only see asterisks or dots no matter what characters are typed in.
radio - to allow the user to select one of severl choices
range - to allow the user to enter a numeric range
reset - used to display a reset button. Once the user clicks on this button, all the fields. Often displayed at the bottom of the form.
search - to allow the user to enter search keywords used to perform a search
submit - to display a submit button. Once the user clicks on this button, the data in the form is transferred to the server. Often displayed at the bottom of the form.
tel - to allow the user to enter a telephone number
text - to allow the use to enter text
in the form are cleared.
time - to allow the user to enter a time by hour, minute, seconds and milliseconds.
week - to allow the user to enter a week

Checkbox fields

Checkbox fields are created like this:
<input type="checkbox" name="checkboxFieldName" value="initialValue" /> Description of checkbox value.

Checkboxes are used to give the user non-exclusive choices. The user can select one or more of the choices presented.

In addition to the attributes all input fields share, the checkbox field also has checked, readonly, and value attributes.

Hidden fields

Hidden fields are used to place invisible input text fields in the form. When the form is submitted, the value of the hidden fields are sent also. They are often used in multiple page forms, where a user starts entering a data in a form, clicks a next button, and enters more data. The data from the first page is inserted into the second page via hidden fields.

A hidden field is created like this:
<input type="hidden" name="myFieldName" value="valueForMyHiddenField" />

Image field

The image field (not the img element, we're talking about the input element with type="image" here) is used to display an image, but when clicked on behaves like a submit button.

An image field is created like this:
<input type="image" src="pathToImageFile" name="nameForImageField" />
This will create a button which appears as an image, which when clicked submits the form.

The image field attributes commonly used are alt, src, width, height, formaction, formenctype, formmethod, formnovalidate, and formtarget.

List fields

Creating a list field also means creating a datalist element. A list field is created like this:
<input type="list" list="idOfDataListElement" name="nameForListField" />

<datalist id=idOfDataListElement>
<option value="value01">
<option value="value02">
<option value="value03">
<option value="value04">
</datalist>
This will create a text field in which the user can select an option from a dropdown list, or whatever value the user types in. The datalist that the list filed can contain any number of options.

A list field, like the select element, will give the user a dropdown list of elements from which to choose. However a list field, unlike a select element, will also accept whatever the user types in.

Password fields

Password fields are like regular text fields in that they allow the user to enter text, but the characters the user types in are now shown. Instead an asterisk or dot is usually displayed for each character typed in. So when the user is in a login page, anyone standing by cannot see what the user typed in for the password. (Of course, that doesn't prevent any person standing by from looking at what the user types on the keyboard.)

A password field is created like this:
<input type="password" name="nameForMyPasswdField" />

Range fields

A range fields allows the user to set a value from within a given range, kind of like sliding a knob from left to right. It also allows you to specify how large a jump occurs as you move the sliding knob from left to write (via the step attribute). However, the range field does not show you the min value, nor max value, nor the value that is currently selected via the slider position. So I find that to make best use of a range field is to use it with JavaScript code. A webpage that discusses using range fields with JavaScript is https://www.geeksforgeeks.org/creating-range-slider-html-using-javascript/.

Attributes asvailable with the range field are value, min, max, and step. The value attribute is used to specify a default value. The min attribute is used to set the minimum value the user can select. The max attribute is used to set the maximum value the user can select. The step attribute is used to set the amount the currently value increases or decreases by. For example, if the current value is 50, and the step is 5, the current value can increase to 55, or decrease to 45, etc.

Text fields

A text field is used to allow the user to enter text data in a form. What type of text data? Well things like a first name, last name, middle name, street address, bank account number, etc. It can be used to allow the user to enter a number, but a number field is now recommended for that. It can also be used to allow the user to enter an e-mail address, but the e-mail field is now recommended for that. In addition, it can be used to allow the user to enter a telephone number, but the telephone field is now recommended for that.

A text field is created like this:
<input type="text" name="nameForTextField" />

Attributes commonly used with text fields are name, value, size, minlength, maxlength and required. The size attribute tells the browser how wide to make the input field when it is displayed. The maxlength attribute tells the browser the maximum number of characters the user is allowed to enter in the input field. Of course, if you supply both minlength and maxlength attributes, the value of the minlength attribute should be less than or equal to the value of the maxlength attribute. The minlength attribute indicates the user must enter at least that many characters. The required attribute, if specified, indicates the user must provide a value in the field.
