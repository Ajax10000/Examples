The Form Tags

This is a practice file - it will not be turned in nor graded. It is for you to learn from and experiment with. Many of the fields in the example forms here do not contain labels. Consider it a challenge to add labels to these fields. This webpage does not use any CSS. Consider it a challenge to add CSS to give this webpage a better "Wow!" factor.

Form-related tags are used to create forms, wherein people can enter data into fields for submission to a web server or be processed by a JavaScript program in the web browser. The form-related tags consist of:
form - starts a form
button - can be used to implement a submit or reset button, or just a button that executes JavaScript code when clicked.
datalist - used with input fields of type="list"
fieldset - a fieldset is used to group one or more input fields. For example, you may wish to group the input fields that provide an address.
input - starts an input field. The type of input field created depends on the type attribute used.
label - used to provide a label to the various input fields (including select tags and textarea tags). Note that option, datalist, fieldset and legend tags do not use labels. Also, the input type="submit" and input type="reset" buttons are typically not given labels. Their value attribute serves as their label.
legend - a legend is used to give a fieldset a title
option - used to provide the options inside a select listbox.
select - creates a drop-down listbox.
textarea - somewhat similar to the input type="text" field, but a textarea lets you enter much more data.

It is common to use the name attribute with the input, select, and textarea fields. This makes it easier to extract specific field information when it is processed in the code that runs on the server.

The input, select, textarea tags will also accept the readonly attribute, as well as the disabled attribute. The readonly attribute is used when you want the user to see what is currently in the field, but not be able to change it. The disabled attribute is used when you want to prevent the user from entering any data in the field (most likely because that field does not apply to the user.)

Form element

A form element is used to create a grouping of input fields whose data will all be submitted to the same server when the form is submitted. A form starts with the <form> tag, and ends with the closing </form> tag. In between are various input fields, labels, fieldsets, legends, textarea and select elements. The data provided in these elements is sent off when the user clicks on the submit button.

Two attributes often used with a form whose associated data is to be submitted to a server are the method and the action attributes. Common values for the method are "post" and "put". Post is often used when submitting data that will be stored or processed on the server. Post is often used when submitting a file that will be stored at the server. For more information on the difference between post and put, see here. If the submit method simply calls a JavaScript function, then there is no need for the action and method attributes in the form element. JavaScript contains functions that allows the webpage developer to read the values in the various input fields.

Button Element

An button field is created like this:
<input type="button" type="typeValue" name="nameForButtonField" />
Possible values for the type attribute are submit, reset, and button. If none is specified, and the button appears within form tags, the default value for the type attribute is submit.

A common attribute for a button that calls a JavaScript function when it is clicked is the onclick attribute.

Select Element

A select element will, by default, only allow you to select one option. You can change that with the multiple attribute. Most browsers will only show one option of a select list. You can change that with the size attribute.

Texarea Element

A textarea field is created like this:
<textarea name="nameForTextareaField" cols="numOfColumns" rows="numOfRows" /> </textarea>
This will create a rectangular area with the indicated number of columns and number of rows, in which the user can enter text.

Common attributes used with the textarea tag are the rows attribute (which indicates how many rows the textarea should span), the cols attribute (which indicates how many columns the textarea should span).

Examples that use textarea element
