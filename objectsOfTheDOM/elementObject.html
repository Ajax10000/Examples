The Element Object

The web browser loads up your webpage and from it creates a DOM tree. For each element in your webpage, it creates an Element object. Like many objects, this Element object comes with properties and methods. We explore these propertie and methods in this webpage.

The HTMLElement object inherits from the Element object. Many element specific objects, like HTMLDivElement, HTMLLIElement, HTMLAnchorElement, HTMLBodyElement, etc., inherit from the HTMLElement object.

Remember that the Element object itself inherits from the Node object.

Properties

childNodes

Access: read only

childNodes is a NodeList object which contains the children of the given element.

Webpages on the Element.childNodes Property

  • https://developer.mozilla.org/en-US/docs/Web/API/Node/childNodes (recall that the Element object inherits from the Node object)
  • https://www.w3schools.com/jsref/prop_node_childnodes.asp
  • https://parrot-tutorial.com/jsref/elem_childnodes.html

children

children is a property of type HTMLCollection. Unlike the childNodes property which contains all nodes (including text and comment nodes), the children property contains only element nodes.

Webpages on the Element.children Property

  • https://www.w3schools.com/jsref/prop_element_children.asp
  • https://www.codespeedy.com/get-child-element-using-javascript-children-property/
  • https://parrot-tutorial.com/jsref/elem_children.html

innerHTML

Access: read/write

The innerHTML property is of type DOMString (which is the same as a regular JavaScript string). It is used to get or set the HTML within a parent element. If it is used to set the HTML within a parent element, the children of the parent element are replaced with the new nodes that are generated by the HTML contained in the innerHTML string property.

Webpages on the Element.innerHTML Property

  • https://developer.mozilla.org/en-US/docs/Web/API/Element/innerHTML
  • https://www.w3schools.com/jsref/prop_html_innerhtml.asp
  • https://parrot-tutorial.com/jsref/elem_innerhtml.html

lastElementChild

Access: read only

The lastElementChild property is of type Node. For a given "parent" element, lastElementChild represents the last child element of that parent element. If the parent element has no children, then lastElementChild will be null.

Methods

getAttribute()

Arguments: 1, a string containing the attribute name whose value is to be returned

Call syntax:
element.getAttribute(attributeName);

Given the attribute name of an element, getAttribute returns the value of the attribute as a string.

It is recommended that you call the Element.hasAttribute() method first to make sure that the element has the given attribute.

getElementsByTagName()

Arguments: 1, a tag name (like "p", "div", "input", "form", etc.)

Call syntax:

Give a tag name, the element.getElementsByTagName method returns a live HTMLCollection of all children of element which have the given tag name.

hasAttribute()

Arguments: 1, an attribute name

Call syntax:
element.hasAttribute(attributeName);

Given an attribute name, element.hasAttribute(attributeName) returns true if element has an attribute with the given attribute name. It returns false if element has no attribute with the given attribute name.

hasAttributes()

Arguments: 0

Call syntax:
element.hasAttributes();

The element.hasAttributes() returns a boolean value. It returns true if element has any attributes. It returns false if it has no attributes.

querySelector()

Arguments: 1, a string indicating one or more CSS selectors to match against.

Call syntax:
element.querySelector(cssSelectorString);

Given a CSS selector string, querySelector searches the descendant elements of the current element and returns the first descendent element that matches the CSS selector provided. If no element is found that matches the given CSS selector, the value null is returned.

Webpages on the Element.querySelector() Method

  • https://developer.mozilla.org/en-US/docs/Web/API/Element/querySelector
  • https://www.w3schools.com/jsref/met_element_queryselector.asp

querySelectorAll()

Arguments: 1, a string indicating one or more CSS selectors to match against.

Call syntax:
element.querySelector(cssSelectorString);

Given a CSS selector string, querySelector searches the descendant elements of the current element and returns all the descendent element that match the CSS selector provided in a (static) NodeList. If no element is found that matches the given CSS selector, the value null is returned. If the string argument provided is not a valid CSS selector string, a SyntaxError occurs.

removeAttribute()

Arguments: 1, the name of the attribute to remove

Call syntax:
element.removeAttribute(attributeName);

Given an attribute name, element.removeAttribute() will remove the attribute from the element.

setAttribute()

Arguments: 2
The first argument is a string indicating the name of the attribute to set. The second argument is the value of the attribute.

Call syntax:
element.setAttribute(attributeName, attributeValue);

The setAttribute method is used to set an attribute of an element. If the attribute already exists, it is updated. If the attribute does not already exist, it is added with the given value.
