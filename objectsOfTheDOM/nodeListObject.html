The NodeList Object

The Document object has several methods. Several of these methods return a NodeList object. For example, the getElementsByTagName, and querySelectorAll, methods return a NodeList. Here we discuss the properties and methods of the NodeList object.

A NodeList is similar to an array. An array stores a collection, and so does a NodeList. We call the elements of a NodeList nodes, because it stores a list of Node objects. Like an array, the NodeList has a length property. Like an array, you can access the nodes of a NodeList with an index. You use the item method of NodeList to get a node from the NodeList: NodeList.item(index); Like an array, the indices of a NodeList are 0-based. To retrieve a node from the NodeList, you can also use the same notation you would for an array: NodeList[index].

Differences are noted between live and static NodeLists. A live NodeList means that if the underlying DOM changes, so will the NodeList. A static NodeList will not change if the underlying DOM changes.

Properties

length

Access: read only

Every NodeList object has a length attribute, a number indicating how many nodes are stored in the NodeList.

Methods

entries()

Arguments: 0

Call syntax:
nodeList.entries();

The entries method returns an iterator that you can use in a for of loop. 

        // The childNodes property is an object of type NodeList
        var nodeList = document.body.childNodes;
        for (var currNode of nodeList.entries()) {
            console.log("currNode = " + currNode);
        }

forEach()

Arguments: 2
The first argument is a callback function which is called once with each node in the NodeList. This callback function shoud take 3 arguments, of which only the first is required, the remaining two being optional.
The second argument is the value to use for this when calling the callback function.

Call syntax:
nodeList.forEach(funcn, thisValue)

As mentioned above, the callback can take up to 3 arguments. The first is the current node in the NodeList being processed. The second is the index of the current node being processed. The third is the NodeList whose nodes are being processed.

item()

Arguments: 1, the index of the item in the NodeList to return

Call syntax:
